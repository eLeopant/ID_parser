# 身份证多场景三分类 —— 基于手写 RBF 核 SVM 的活体检测系统

## 项目简介

本项目针对**身份证活体检测**这一复杂工程问题，构建了一套完整的**三分类（实拍 / 拍屏 / 复印件翻拍）**机器学习系统。核心分类器为 **100% 手写 RBF 核支持向量机（SVM）**，采用 SMO 算法求解对偶问题、OvO 策略实现多分类，不依赖任何第三方 SVM 库。

## 快速开始

### 1. 安装依赖

```bash
# Python >= 3.8
pip install -r requirements.txt
```

### 2. 一键运行全流程（预处理 → 训练 → 测试）

```bash
# 三分类模式（默认）
python main.py

# 二分类模式（实拍 vs 非实拍）
python main.py --binary_mode

# 自定义超参数搜索空间
python main.py --C_list 0.1 1 10 100 --gamma_list 0.001 0.01 0.1
```

### 3. 单独运行各模块

```bash
# 仅预处理
python pre_process.py

# 仅训练（需要先完成预处理）
python train.py

# 仅测试评估
python test.py

# 活体检测推理（对 task/task.jpg 预测）
python predict.py
python predict.py --image task/task.jpg --model outputs/best_model.pkl
```

### 4. 断点复用

```bash
# 跳过预处理（直接使用已有 outputs/*.pkl）
python main.py --skip_preprocess

# 跳过预处理与训练（直接测试已有模型）
python main.py --skip_preprocess --skip_training
```

## 核心技术要点

### 零数据泄露
- **按身份证 ID 颗粒度**划分训练集（前 80% ID）与测试集（后 20% ID）
- 同一张身份证的所有切块**不会同时出现在训练集与测试集**中

### 三类手工纹理特征（8191 维总特征）
| 特征 | 维度 | 作用 |
|------|:----:|------|
| Uniform LBP (P=8, R=1) | 59 | 捕捉拍屏摩尔纹、复印件纸张粗糙度 |
| HOG (orientations=9) | 8100 | 捕捉碳粉扩散、边缘锯齿感 |
| HSV 颜色直方图 (H+S) | 32 | 捕捉屏幕蓝光偏色、复印件低饱和度 |

### 100% 手写 SVM
- **核函数**：RBF（高斯核）$K(x_i, x_j) = \exp(-\gamma \|x_i - x_j\|^2)$
- **求解算法**：SMO（序列最小优化），支持软间隔惩罚参数 C
- **多分类**：OvO（一对一）策略，训练 $C_3^2 = 3$ 个二分类器，投票决策
- **超参数搜索**：手动封装网格搜索 + 5 折交叉验证（按 ID 维度分层）
- **搜索空间**：$C \in \{0.1, 1, 10, 100\}$，$\gamma \in \{0.001, 0.01, 0.1, 1\}$

### 活体检测部署
- `predict.py` 加载训练好的模型，对单张身份证照片进行推理
- 输出 `task/is_real.yaml`，格式简洁：`is_real: 1`（实拍）或 `is_real: 0`（非实拍）
- 可被上游边缘检测模块作为子流程调用

## 实验结果速览

### 三分类（测试集，严格 ID 隔离）

| 指标 | 实拍 (Real) | 拍屏 (Screen) | 复印件 (Paper) |
|------|:-----------:|:-------------:|:--------------:|
| Precision | 0.3906 | 0.0000 | 0.9296 |
| Recall | 0.9091 | 0.0000 | 0.7500 |
| F1-Score | 0.5464 | 0.0000 | 0.8302 |

- **整体准确率**：58.29%
- **交叉验证最优准确率**：59.43%（C=10, γ=0.001）

> **注**：拍屏类 0% 识别率源于训练样本极少（7张原图/135个切块），且拍屏与实拍纹理在高压缩 JPEG 下差异微弱。这是该复杂工程问题的核心难点之一。

## 复杂工程问题特征对照

| 特征 | 本项目体现 |
|------|-----------|
| ① 需深入工程原理分析 | 手工纹理特征工程（LBP/HOG/HSV）的数学原理与选择 |
| ② 多方面技术/非技术因素 | 光学成像（摩尔纹）、纸张物理特性、颜色科学 |
| ③ 需建立抽象模型 | 256×256 微观纹理切块建模，消除全局语义干扰 |
| ④ 非仅靠常用方法 | 100% 手写 SMO 求解器 + OvO 多分类策略 |
| ⑤ 因素不完全包含在规范中 | 身份证材质多样性、拍摄设备差异无统一标准 |
| ⑥ 各方利益不完全一致 | 安全风控（严格检测）vs 用户体验（低误拒率） |
| ⑦ 高度综合性 | 预处理→特征工程→SVM 求解→超参数优化→部署推理 |
