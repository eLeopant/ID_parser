# 卡片边缘检测系统

基于 KNN 机器学习方法和 OpenCV 传统方法的卡片边缘检测项目。

## 功能特点

- **两种检测方法**：KNN（机器学习）和 OpenCV（传统图像处理）
- **绿色线条标记**：检测到的卡片边缘用绿色线条框出
- **自动裁剪**：输出裁剪后的卡片图片（透视变换校正）
- **支持重复训练测试**：KNN 模型可多次训练和测试
- **终端输出**：显示识别成功率和正确率

## 项目结构

```
Edge Detection/
├── main.py                   # 主程序入口
├── knn_card_detection.py      # KNN 检测模块（需训练）
├── opencv_edge_detector.py    # OpenCV 检测模块（无需训练）
├── label_tool.py             # 角点标注工具
├── requirements.txt           # 依赖包
└── data/
    ├── train/
    │   ├── images/           # 训练集图片
    │   └── labels/           # 训练集标签
    ├── test/
    │   ├── images/           # 测试集图片
    │   ├── labels/           # 测试集标签（可选）
    │   └── output/           # 检测结果输出
    └── models/               # 训练好的模型
```

## 安装

```bash
pip install -r requirements.txt
```

## 使用方法

```bash
python main.py
```

| 选项 | 功能 | 说明 |
|------|------|------|
| 1 | 训练 KNN 模型 | 需要准备训练数据 |
| 2 | KNN 方法测试 | 使用训练好的模型检测 |
| 3 | OpenCV 方法测试 | 无需训练，推荐使用 |

## 数据准备

### 训练集（KNN 方法）

**训练图片目录**：`data/train/images/`

放置原始卡片图片（.jpg, .png 格式）。

**训练标签目录**：`data/train/labels/`

为每张图片创建同名 .txt 文件，标签格式：

```
x1,y1 x2,y2 x3,y3 x4,y4
```

示例（归一化坐标 0-1）：
```
0.1,0.1 0.9,0.1 0.9,0.9 0.1,0.9
```

坐标顺序：左上 → 右上 → 右下 → 左下

### 测试集

**测试图片目录**：`data/test/images/`

放置待检测的卡片图片。

**测试标签目录**：`data/test/labels/`（可选）

如果有标签文件，程序会自动计算识别准确率。

## 输出结果

测试完成后，在 `data/test/output/` 目录生成：

```
data/test/output/
├── 1.png              # 带边框的检测图
├── cropped_1.png      # 裁剪后的卡片图
├── 2.png
├── cropped_2.png
└── ...
```

终端会显示：


## 两种方法对比

| 特性 | KNN 方法 | OpenCV 方法 |
|------|----------|-------------|
| 是否需要训练 | 是 | 否 |
| 训练数据量 | 建议 20+ 张 | 不需要 |
| 泛化能力 | 依赖训练数据 | 通用 |
| 检测速度 | 较快 | 快 |
| 鲁棒性 | 中等 | 好 |

**推荐**：首次使用或无训练数据时，使用 OpenCV 方法。

## 标注工具

如果需要手动标注训练数据的角点，可使用：

```bash
python label_tool.py
```

操作方法：
1. 点击卡片的四个角点（左上、右上、右下、左下）
2. 按 `s` 保存标签
3. 按 `n` 下一张图片
4. 按 `q` 退出
